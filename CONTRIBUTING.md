# 🤝 Contributing Guide - 贡献指南

感谢您对 DailyClaw 的兴趣！以下是参与项目的方法。

## 🐱 六猫协作规则

DailyClaw 采用独特的 **六猫协作系统**。当你参与贡献时，可以选择扮演以下角色：

| 角色 | 适合人群 | 职责 |
|------|----------|------|
| 🎀 **Athena** | 架构师、技术负责人 | 设计系统架构 |
| 🐯 **Hephaestus** | 开发者、工程师 | 编写实现代码 |
| 🔵 **Apollo** | 测试工程师、QA | 测试验证 |
| 🧶 **Hermes** | DevOps、集成工程师 | 系统集成 |
| ⚫ **Artemis** | 安全工程师、SRE | 安全审查 |

## 🚀 如何贡献

### 1. 提交 Issue

发现 bug 或有新想法？先创建 Issue：

- 使用清晰的标题
- 描述问题/建议
- 标注相关标签

### 2. Fork & PR

```bash
# Fork 仓库
# 克隆你的 fork
git clone https://github.com/YOUR_USERNAME/dailyclaw.git

# 创建分支
git checkout -b feature/your-feature

# 提交更改
git commit -m "🐱 添加新功能"

# 推送到你的 fork
git push origin feature/your-feature

# 创建 Pull Request
```

### 3. 提交规范

使用 emoji 前缀：

| Emoji | 类型 | 示例 |
|-------|------|------|
| ✨ | 新功能 | `✨ 添加爬虫模块` |
| 🐛 | Bug 修复 | `🐛 修复登录问题` |
| 📚 | 文档 | `📚 更新 README` |
| 🐱 | 六猫相关 | `🐱 优化协作流程` |
| ⚡ | 性能优化 | `⚡ 提升查询速度` |

### 4. 代码审查

PR 需要：
- 至少 1 只猫 🐱 的审查
- 通过自动化测试
- 符合项目规范

## 📁 目录结构

```
dailyclaw/
├── docs/           # 文档
├── evolution/      # 进化记录（自动生成）
├── modules/        # 功能模块
├── showcase/       # 成果展示
└── .github/        # GitHub 配置
```

## 🎯 贡献方向

我们特别欢迎以下方面的贡献：

- 📚 **文档改进** - 帮助完善中英文文档
- 🎪 **演示案例** - 分享你的使用场景
- 🔧 **工具集成** - 开发新的 MCP 工具
- 🐛 **Bug 修复** - 发现问题并修复
- 💡 **新功能** - 提出并实现新想法
